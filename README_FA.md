## این چیست؟

Lapka SMS یک اپلیکیشن کامل SMS با رمزنگاری داخلی پیام‌ها است. پیام‌های رمزنگاری‌شده با استفاده از پنهان‌نگاری کدگذاری می‌شوند — برای هر کسی که آنها را رهگیری کند، مانند متن سیریلیک یا کلمات روسی به نظر می‌رسند. دشمن اصلی **اپراتور تلفن همراه** است که می‌تواند محتوای SMS را در حین انتقال بخواند.

> **هر دو طرف به Lapka SMS** با کلید رمزنگاری یکسان نیاز دارند تا پیام‌های رمزنگاری‌شده رد و بدل کنند. SMS معمولی با هر تلفنی کار می‌کند.

## امکانات

### رمزنگاری ([جزئیات پروتکل](PROTOCOL_FA.md))
- **AES-256-GCM** — رمزنگاری احراز هویت‌شده با استخراج کلید HKDF-SHA256
- **محافظت از بازپخش** — پیام‌های قدیمی و تکراری رد می‌شوند
- **پنهان‌سازی طول پیام** — padding از تحلیل ترافیک جلوگیری می‌کند

### طرح‌های پنهان‌نگاری
| طرح | نمونه خروجی | مناسب برای |
|---|---|---|
| Base64 | `dGVzdA==` | عمومی، فشرده |
| Base64 سیریلیک | `дГВздА==` | ترکیب با متن سیریلیک |
| کلمات روسی | `молоко дерево книга` | شبیه زبان طبیعی |

### مدیریت کلید
- **کلید برای هر مکالمه** — کلید متفاوت برای هر مخاطب
- **کلید سراسری** — کلید پیش‌فرض برای همه مکالمات
- **اشتراک‌گذاری QR کد** — اسکن برای تبادل کلید
- **اثر انگشت SHA-256** — تأیید اصالت کلید
- **Android Keystore** — ذخیره‌سازی کلید با پشتیبانی سخت‌افزاری
- **EncryptedSharedPreferences** — کلیدها در حالت استراحت رمزنگاری شده‌اند

### حریم خصوصی و امنیت
- **FLAG_SECURE** — مخفی‌سازی محتوای برنامه در تعویض‌کننده وظایف (قابل تنظیم)
- **حذف خودکار پیام‌های رمزنگاری‌شده** — تایمر قابل تنظیم
- **SMS برای بازنشانی** — دریافت یک SMS از پیش تعیین‌شده همه کلیدهای رمزنگاری را پاک و تنظیمات رمزنگاری را بازنشانی می‌کند. پیام‌ها باقی می‌مانند، اما غیرقابل رمزگشایی می‌شوند
- **بدون تحلیل و ردیابی**
- **پایگاه داده رمزنگاری‌شده Realm**

### اپلیکیشن SMS
- رابط کاربری Material Design با تم‌های قابل تنظیم
- حالت شب (خودکار/دستی/سیستم)
- تنظیمات اعلان برای هر مکالمه
- ارسال با تأخیر
- گزارش تحویل
- پشتیبانی از دو سیم‌کارت
- عملیات کشیدن
- بیش از ۴۰ زبان

## مدل تهدید

| محافظت‌شده | محافظت‌نشده |
|---|---|
| محتوای پیام (AES-256-GCM) | فراداده‌های ارتباطی (چه کسی، چه زمانی، چند بار) |
| طول پیام (padding) | استفاده هر دو طرف از Lapka SMS |
| مواد کلید در ذخیره‌سازی (Keystore + EncryptedSharedPreferences) | دسترسی فیزیکی به دستگاه باز شده |
| محتوای برنامه در تعویض‌کننده وظایف (FLAG_SECURE) | امنیت دستگاه گیرنده |

## شروع کار

### ۱. نصب

آخرین APK را از [Releases](https://github.com/GeorgiyDemo/Lapka-SMS/releases) دانلود و نصب کنید. هنگام درخواست، به عنوان اپلیکیشن پیش‌فرض SMS تنظیم کنید.

### ۲. راه‌اندازی رمزنگاری

۱. **تنظیمات** ← **تنظیمات کلید رمزنگاری** را باز کنید
۲. برای ایجاد کلید AES-256 روی **تولید کلید** ضربه بزنید
۳. کلید را با مخاطب خود از طریق **QR کد** به اشتراک بگذارید (حضوری ملاقات کنید یا از کانال امن استفاده کنید)
۴. مطمئن شوید که **اثر انگشت کلید** در هر دو دستگاه مطابقت دارد
۵. یک **طرح کدگذاری** انتخاب کنید (Base64 / Base64 سیریلیک / کلمات روسی)

### ۳. ارسال پیام‌های رمزنگاری‌شده

فقط مانند همیشه پیام بفرستید. اگر رمزنگاری برای یک مکالمه فعال باشد، پیام‌ها به طور خودکار رمزنگاری و کدگذاری می‌شوند. پیام‌های رمزنگاری‌شده ورودی به طور شفاف رمزگشایی می‌شوند.

## ساخت از سورس

نیاز به **JDK 17**.

```bash
git clone https://github.com/GeorgiyDemo/Lapka-SMS.git
cd Lapka-SMS
./gradlew assembleDebug
```

## معماری

```
presentation/   لایه رابط کاربری اندروید (Activities, Conductor Controllers, ViewModels)
domain/          منطق کسب‌وکار، تعامل‌کننده‌ها، مدل‌ها
data/            مخازن، دریافت‌کننده‌ها، پایداری Realm
common/          ابزارهای مشترک
psms-lib/        کتابخانه رمزنگاری (AES-GCM, HKDF, کدگذاری‌کننده‌های پنهان‌نگاری)
android-smsmms/  فریمورک قدیمی MMS/SMS
```

الگوهای کلیدی:
- **Conductor** — ناوبری (Controllers داخل Activities)
- **Dagger 2** — تزریق وابستگی
- **RxJava 2** + AutoDispose — جریان‌های واکنشی
- **Realm** — ذخیره‌سازی رمزنگاری‌شده داده‌ها

## تفاوت با upstream

Lapka SMS یک فورک از [Partisan-SMS](https://github.com/wrwrabbit/Partisan-SMS) (که خود فورکی از [QKSMS](https://github.com/moezbhatti/qksms) است). تغییرات در Lapka SMS:

- ارتقای پروتکل رمزنگاری به نسخه ۳ با کدگذاری‌کننده‌های پنهان‌نگاری جدید
- ارتقای وابستگی‌ها (Dagger 2.52, Glide 4.16, Kotlin 1.9, compileSdk 35)
- ذخیره‌سازی رمزنگاری‌شده پایگاه داده (رمزنگاری Realm از طریق Android Keystore)
- EncryptedSharedPreferences برای مواد کلید
- تأیید اثر انگشت کلید
- انتخاب زبان داخل برنامه
- تقویت امنیت (تنظیمات امنیت شبکه، لاگ‌گذاری خصوصی، FLAG_SECURE)
- خط لوله CI/CD
- مدرن‌سازی کد (پاکسازی API منسوخ، مهاجرت به AndroidX)
